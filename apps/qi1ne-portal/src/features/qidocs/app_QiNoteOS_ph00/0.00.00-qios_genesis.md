### QiRealms
Meaning universes. A Realm answers **"who/what universe does this belong to?"** not "what file type is it." Realms are boundaries of context and ownership.
“Realms are pre-allowed by registry and may be enabled/disabled without schema migration.”

Typical Realm roots:
- **QiPersonal** (your private universe)
- **QiBusiness** (QiAlly/Kiali operations)
- **QiClients** (each client’s universe)
- **QiVault** (knowledge universe)
- **QiPlayground** (Catchall Sandbox)
- **QiResearch** (source material, references)
- **QiTemp**
- **Archives** (retired/cold universes)

Realm examples:
- A personal medical PDF belongs to **QiPersonal**.
- A PunchPro contract belongs to **QiClients/punchpro**.
- A QiOS spec belongs to **QiVault**.

### QiAssets
Files that are _not primarily knowledge text_ but still carry meaning. Assets are stored **inside Realms** because their context depends on ownership.

- **Video:** `.mp4 .mov .mkv .webm .avi`

Asset placement examples:
- Brand logo → `realms/qibusiness/assets/images/branding/`
- Client before/after photos → `realms/qiclients/<slug>/assets/images/`
- Sora ad video renders → `realms/qibusiness/assets/videos/ads/`

## System Files (Root Level)
QiOS recognizes and protects essential system files at the repository root. These files are part of the infrastructure and should not be processed, moved, or deleted by GINA:

**Version Control & Git:**
- `.git/` - Git repository metadata (protected, ignored by GINA)
- `.gitignore` - Git ignore patterns (protected, ignored by GINA)

**Environment & Configuration:**
- `.env` - Environment variables (protected, ignored by GINA, never committed)
- `env.example` - Environment variable template (protected, safe to commit)

**Editor & Tooling:**
- `.obsidian/` - Obsidian vault configuration (protected, ignored by GINA)
- `.cursorrules` - Cursor AI editor rules (protected, governs AI behavior)

**Visual Planning:**
- `*.canvas` - Canvas files for visual planning (protected, recognized as system files)
- `*kanban*.md` - Kanban board files (protected, recognized as system files)

These files are protected by Layer 1 (Dark Matter Physics) and are exempt from:
- Semantic routing
- Front matter requirements
- Naming law enforcement (they follow tool conventions)
- Automatic deletion or quarantine

GINA recognizes these as infrastructure and leaves them untouched.
